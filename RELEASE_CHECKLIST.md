# 🚀 Focus Flow 출시 체크리스트 (Release Checklist)

출시 전, 아래 항목들을 반드시 순차적으로 확인하고 업데이트합니다.

## 1. 버전 및 정보 업데이트
- [ ] **build.gradle.kts**: `versionCode` 상향 및 `versionName` 최신화.
- [ ] **README.md**: 최신 버전 정보 업데이트 및 **이전 버전의 주요 기능을 지우고 이번 버전의 새로운 기능(NEW/FIX)**을 작성.
- [ ] **UI 표시 확인**: 스플래시 화면, 설정 탭 등 UI 내 버전 정보가 최신인지 확인.

## 2. 릴리즈 노트 작성 (⚠️ 중요: 이전 내용 복사 금지)
- [ ] **변경 로그 분석**: 지난 태그 이후의 Git 로그를 기반으로 실제 추가/수정된 사항 정리.
- [ ] **배포용 문구 작성**: 사용자가 읽기 쉬운 이모지와 함께 **이번 릴리즈의 핵심 변경점**을 5~7개 내외로 요약.
- [ ] **설치 안내**: APK 다운로드 및 업데이트 방법 포함.

## 3. 코드 및 안정성 확인
- [ ] **디폴트 값 확인**: `SettingsRepository` 등의 기본값이 기획 의도와 일치하는지 확인.
- [ ] **영속성**: 앱 업데이트 시 설정 데이터 유지 로직 검증.
- [ ] **시스템 채널**: 알림 채널 설정(v3)이 의도대로 시스템 소리를 차단하는지 확인.

## 4. Git 및 배포
- [ ] **Git Push**: 모든 변경 사항 푸시.
- [ ] **Git Tag**: 버전 네임 태그 생성 및 푸시 (`git tag v1.x.x`).
- [ ] **Github Release**: 태그에 맞춰 새 릴리즈를 생성하고, **2번에서 작성한 배포용 문구**를 본문에 삽입 후 APK 업로드.

---
*이 체크리스트는 모든 릴리즈 작업의 기본 지침이며, 매 버전마다 "신선한" 정보를 제공해야 합니다.*
